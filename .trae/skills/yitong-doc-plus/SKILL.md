# 亿通文档格式转换技能

本技能用于将文档转换为符合亿通公司标准研发文档格式的输出。

## 核心功能

1. 识别文档类型并选择对应的亿通文档模板
2. 按照亿通文档规范重组文档结构
3. 应用标准的表格样式和格式
4. 确保包含必要的图表辅助说明

## 使用流程

1. 分析用户需求，确定目标文档类型
2. 读取并理解原始文档内容
3. 根据亿通文档规范重组内容
4. 应用标准表格样式
5. 输出符合规范的文档

## 文档类型

亿通公司包含5种标准文档类型：

| 文档类型 | 命名规范 | 主要用途 |
|---------|---------|---------|
| 会议纪要 | YYYYMMDD会议纪要 | 记录会议内容和决议 |
| 软件设计说明书 | 软件设计说明书XXX | 详细设计文档 |
| 软件系统技术方案 | 软件系统技术方案模板 | 技术方案汇报 |
| 需求分析说明书 | 系统需求分析说明书XXX | 需求规格说明 |
| 需求变更文档 | YYYYMMDD-需求描述 | 需求变更记录 |

详细的文档结构和规范请参考 references/yitong-doc-guidelines.md

## 表格样式规范

### 标准表格样式

所有表格使用 Markdown 表格格式，遵循以下规范：

**示例1：字段说明表**
| 字段 | 说明 |
|-----|------|
| 会议主题 | 会议的核心议题 |
| 会议时间 | 格式：YYYY-MM-DD HH:MM-HH:MM |
| 会议地点 | 会议召开地点 |

**示例2：里程碑表格**
| 里程碑阶段 | 时间节点 | 核心交付物 |
|-----------|---------|-----------|
| 需求确认 | YYYY-MM | 需求说明书（简版） |
| 技术验证 | YYYY-MM | 技术原型demo + 验证报告 |

**示例3：需求变更文档头部表格（特殊4行格式）**
| 项目名称 | (空) |
|---------|------|
| 变更提出方 | 监管单位/业务/企划/平台等 | 变更提出日期 | YYYY-MM-DD |
| 建议完成时间 | YYYY-MM-DD | 实际完成时间 | YYYY-MM-DD |
| 变更原因 | (空) |

### 数据库表设计表格

| 字段名 | 数据类型 | 长度 | 主键 | 外键 | 约束 | 说明 |
|-------|---------|------|-----|------|------|------|
| id | int | 11 | 是 | 否 | 非空、自增 | 主键ID |
| name | varchar | 100 | 否 | 否 | 非空 | 名称 |
| create_time | datetime | - | 否 | 否 | 非空 | 创建时间 |

## 通用编写规范

### 文档结构要求
- 所有文档需包含必要的图表辅助说明
- 使用分点列表（1. 2. 3.）记录内容
- 关键标题使用粗体（**标题**）

### 时间格式
- 日期格式：YYYY-MM-DD
- 时间格式：YYYY-MM-DD HH:MM-HH:MM
- 文件名日期：YYYYMMDD

### 参考资料
所有研发文档可参考以下规范文件：
- 《YT-YF-01-001 软件产品设计开发控制程序（B0）》
- 《需求分析方法指南》
- 《软件开发流程填写规范》
- 《技术安全管理规范》
- 《命名规程》

## 转换步骤

1. **确定文档类型**：根据用户需求确定目标文档类型
2. **提取内容**：从原始文档中提取关键信息
3. **重组结构**：按照亿通文档规范重组内容结构
4. **应用格式**：
   - 使用标准表格样式
   - 统一时间格式
   - 应用标题层级
5. **补充内容**：添加必要的图表提示和参考资料
6. **输出文档**：生成符合规范的最终文档
